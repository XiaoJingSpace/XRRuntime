## 格式规范

### Word 文档格式

- **标题1（第X章）**: 黑体，20pt，加粗，居中
- **标题2（X.X 节）**: 黑体，16pt，加粗，左对齐
- **标题3（X.X.X 小节）**: 黑体，14pt，加粗，左对齐
- **正文**: 宋体，12pt，首行缩进2字符，1.5倍行距
- **代码块**: Courier New，10pt，浅灰背景，左右缩进1cm
- **Mermaid 图表**: 显示为文本说明，需要手动转换为图片
- **列表**: 宋体，12pt，悬挂缩进
- **页面设置**: 上下边距2.5cm，左右边距2cm

### 图表处理

- **Mermaid 图表**: 在 Markdown 中使用 ` ```mermaid ` 代码块
- **截图**: 存放在 `book/images/` 目录，使用 `![描述](images/filename.png)` 引用
- **图表转换**: Mermaid 图表需要转换为图片后插入 Word 文档

### Markdown 格式

- 使用标准 Markdown 语法
- 代码块使用三个反引号
- 列表使用 `-` 或数字编号
- 标题使用 `#` 标记

## 更新文档

### 更新 Markdown 源文件

1. 编辑对应的 `.md` 文件
2. 运行转换脚本生成 Word 文档

### 重新生成 Word 文档

```bash
python scripts/md_to_docx.py
```

### 注意事项

- 确保所有章节文件都存在
- 检查文件路径是否正确
- Word 文档生成时，如果原文件被打开，会保存为临时文件
- 关闭 Word 后，可以手动替换或重命名临时文件
